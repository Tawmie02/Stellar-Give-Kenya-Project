## What's Needed for Full Integration

### 1. Install Soroban CLI
```bash
# Install Soroban CLI
cargo install soroban-cli

# Verify installation
soroban --version
```

### 2. Deploy Contracts to Testnet

#### Build Contracts
```bash
# Build Project Registry
cd skillreceipt/contracts/project-registry
cargo build --target wasm32-unknown-unknown --release

# Build Escrow
cd ../escrow
cargo build --target wasm32-unknown-unknown --release

# Build Receipt
cd ../receipt
cargo build --target wasm32-unknown-unknown --release
```

#### Deploy to Stellar Testnet
```bash
# Set network to testnet
soroban network fetch --global https://soroban-testnet.stellar.org
soroban network apply --global testnet

# Deploy Project Registry
soroban contract deploy \
  --wasm target/wasm32-unknown-unknown/release/project_registry.wasm \
  --source <YOUR_WALLET_ADDRESS>

# Deploy Escrow
soroban contract deploy \
  --wasm target/wasm32-unknown-unknown/release/escrow.wasm \
  --source <YOUR_WALLET_ADDRESS>

# Deploy Receipt
soroban contract deploy \
  --wasm target/wasm32-unknown-unknown/release/receipt.wasm \
  --source <YOUR_WALLET_ADDRESS>
```

**Save the contract addresses returned from deployment!**

### 3. Configure Frontend with Contract Addresses

Update `frontend/src/utils/contractIntegration.ts`:
```typescript
const CONTRACT_ADDRESSES = {
  projectRegistry: 'PROJECT_REGISTRY_CONTRACT_ADDRESS',
  escrow: 'ESCROW_CONTRACT_ADDRESS',
  receipt: 'RECEIPT_CONTRACT_ADDRESS',
};
```

### 4. Update ProjectContext to Use Contracts

Replace mock functions in `frontend/src/context/ProjectContext.tsx` with actual contract calls:

```typescript
import { 
  buildContractTransaction, 
  submitTransaction,
  addressToScVal,
  stringToScVal,
  numberToScVal
} from '../utils/contractIntegration';

// Example: Update createProject to use contract
const createProject = useCallback(async (input: CreateProjectInput) => {
  const args = [
    addressToScVal(input.clientAddress),
    stringToScVal(input.title),
    stringToScVal(input.description),
    numberToScVal(parseInt(input.amount)),
  ];
  
  const tx = await buildContractTransaction(
    CONTRACT_ADDRESSES.projectRegistry,
    'create_project',
    args,
    input.clientAddress
  );
  
  // Sign with Freighter and submit
  // ...
}, []);
```

### 5. Add Contract-Specific Functions

Create contract client functions for each contract operation:

**Project Registry Operations:**
- `create_project`
- `assign_freelancer`
- `mark_completed`
- `mark_paid`
- `get_project`

**Escrow Operations:**
- `deposit` (lock funds)
- `mark_complete`
- `release_payment`
- `get_escrow`

**Receipt Operations:**
- `create_receipt`
- `get_receipt`

### 6. Test End-to-End Workflow

1. **Client Flow:**
   - Connect wallet with Freighter
   - Create project (calls Project Registry contract)
   - Deposit funds to escrow (calls Escrow contract)
   - Review freelancer applications
   - Select freelancer (updates Project Registry)
   - Approve completion (calls Escrow release_payment)
   - Receive receipt (auto-generated by Receipt contract)

2. **Freelancer Flow:**
   - Connect wallet with Freighter
   - Browse projects (reads from Project Registry)
   - Submit application (calls Project Registry)
   - Mark work complete (calls Escrow mark_complete)
   - Receive payment (automatic via Escrow release)
   - View receipt (reads from Receipt contract)

I've created `frontend/src/utils/contractIntegration.ts` with helper functions:

- `buildContractTransaction()` - Build contract call transactions
- `submitTransaction()` - Submit signed transactions
- `simulateContractCall()` - Simulate without submitting
- Type conversion functions (addressToScVal, etc.)

These utilities are ready to use once contracts are deployed and addresses are configured.
